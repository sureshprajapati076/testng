For running in command line via maven.

for demo YoutubeRunner5.java is added with tag that is not in Feature file

also maven surefire plugin is confiugred in pom.xml as below:

<build>

            <plugins>
                <plugin>
                    <groupId>org.apache.maven.plugins</groupId>
                    <artifactId>maven-surefire-plugin</artifactId>
                    <version>3.0.0-M7</version>
                    <configuration>
                        <testClassesDirectory>${project.build.directory}/test-classes</testClassesDirectory>
                        <testFailureIgnore>true</testFailureIgnore>
                        <includes>
<!--                            <include>**/YoutubeRiunner.java</include>-->
                            <include>${buildrunner}</include>
                        </includes>
                        <excludes>
                            <exclude>none</exclude>
                        </excludes>
                        <parallel>methods</parallel>
                        <threadCount>1</threadCount>
                        <perCoreThreadCount>false</perCoreThreadCount>
<!--                        <systemPropertyVariables>-->
<!--                            <cucumber.options>${cucumber.options}</cucumber.options>-->
<!--                            <threadcount>${threadCount}</threadcount>-->
<!--                        </systemPropertyVariables>-->
                    </configuration>
                </plugin>
            </plugins>

    </build>



NOW,

we can run below command in terminal:

mvn test -D"cucumber.filter.tags"="@youtube" -Dbuildrunner="YoutubeRunner5.java"

it will replace buildrunner variable with YoutubeRunner5.java and replace tag with @youtube which is matching with feature file and it runs well.


Running specific xml file from maven:

Easiest solution would be, add the below lines in surefire plugin.

<suiteXmlFiles>
    <!-- pass testng.xml files as argument from command line -->
    <suiteXmlFile>${suiteXmlFile}</suiteXmlFile>
</suiteXmlFiles>
And run your xml file as below,

mvn clean test -Dsurefire.suiteXmlFiles=/path/to/testng.xml
